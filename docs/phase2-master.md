# 二期开发主控文档

## 1. 文档目标

本文档作为二期总控入口，用于统一说明二期开发的目标、边界、优先级和实施顺序。后续开发、测试、验收默认以本文档为总览，以 `phase2-checklist.md` 为执行清单。

## 2. 当前项目所处阶段

截至 2026-06-25：

- 一期主工作流已经可以完整跑通
- Sprint 1～8 核心项已完成（…用户与鉴权、业务校验与导出增强）
- 下一步：§6.3 交互统一、§6.4 远期 AI 能力（按需排期）
- 执行清单见 `phase2-checklist.md` §3 Sprint 迭代计划

## 3. 一期基线能力

当前系统已经具备以下基线：

- 项目、素材、主题、节奏、分镜、导出的完整页面流
- 项目与素材 CRUD
- 主题生成与选定
- 节奏生成与保存
- 分镜生成、编辑、插入、删除、排序
- 导出预览与下载
- 基础登录入口

## 4. 一期基线业务口径

二期开始前，以下口径视为一期既定事实，不再轻易改动：

- 工作流顺序为：新建 → 录入 → 主题 → 节奏 → 分镜 → 导出
- 分镜生成优先参考节奏模块的节拍点
- 一期不支持策略性复用同一素材来补齐时长
- 如确需复用同一源素材，临时方案是人工先切分为多个素材再录入
- 当前分镜排序为“功能标签阶段优先 + 同标签内按素材录入顺序”
- 项目目标时长最小值为 5 秒

## 5. 二期目标

二期核心目标不是重做一期，而是在一期稳定工作流上增强三类能力：

1. 更真实的输入
2. 更智能的建议
3. 更稳的工程结构

对应落地方向：

- 音频上传与真实节拍识别
- LLM 主题、分镜、导出建议深化
- 用户与鉴权体系后端化
- 数据结构与服务分层进一步规范
- 自动化校验与测试补齐

## 6. 二期边界

二期仍然不做以下内容：

- 自动粗剪产出完整成片
- 高复杂度镜头复用策略引擎
- 多角色协同权限体系完整版
- 重型媒体资产管理平台

## 7. 当前优先级

### P0：必须先解决的技术债

- 文档与文本编码统一，避免乱码反复污染
- 关键服务分层整理，减少 `repository` 继续膨胀
- 数据库变更与迁移规范化
- 明确素材排序、分镜排序、状态刷新等关键口径

### P1：优先开发的功能点

- 音频上传与自动节拍识别增强
- LLM 主题建议增强
- LLM 分镜建议增强
- LLM 导出建议增强
- 通用 LLM Provider / Auth / Gateway 体系
- 登录与用户信息后端化

### P2：后续增强项

- 更细颗粒度的业务校验
- 更多导出格式和更强预览体验
- 更稳定的前端状态反馈与交互统一
- 大模型辅助排序与分镜组织策略

## 9. LLM 统一接入原则

二期 LLM 能力统一遵循以下原则：

- 不绑定单一厂商命名
- 优先支持 API Key 模式
- 允许后续接入 OAuth / Device Code
- 不依赖浏览器 Cookie 注入或抓取网页登录态
- Provider 配置、鉴权逻辑、模型调用逻辑分层处理

详细规范见 `phase2-llm-integration-standard.md`。
