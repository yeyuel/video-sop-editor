# 一期实现上下文压缩版

## 1. 文档目的

本文档用于快速说明当前一期已经实现到什么程度、哪些规则已经固化、哪些能力明确留到二期，方便后续开发在不回看全部历史记录的情况下继续推进。

## 2. 产品定位

- 产品名称：旅行短视频剪辑导演助手
- 一期目标：帮助用户把项目、素材、主题、节奏、分镜、导出信息结构化沉淀下来
- 一期核心价值：输出“可执行剪辑方案”，而不是自动产出成片

## 3. 一期已实现范围

当前工作流已经统一为 6 个阶段：

1. 新建
2. 录入
3. 主题
4. 节奏
5. 分镜
6. 导出

当前已落地页面与基础 CRUD：

- 项目列表、新建项目、编辑项目、删除项目
- 素材列表、新增素材、编辑素材、删除素材
- 主题列表、主题生成、主题选择与保存
- 节奏规划生成、节奏编辑与保存
- 分镜列表、分镜生成、单条分镜编辑、删除、插入、排序调整
- 导出预览与下载

## 4. 一期已固化业务规则

### 4.1 项目与素材

- 项目状态默认 `draft`，创建时不要求用户手动选择状态
- 素材 ID 由系统自动生成
- 项目保存素材根目录，素材保存相对路径
- 素材建议时长由用户填写，不是写死值
- 目标时长下限已调整为 5 秒

### 4.2 工作流顺序

- 节奏在分镜之前，因为分镜时间线需要优先参考节拍点
- 导出阶段当前由用户手动整理标题、标签和文案，也支持二期新增的 LLM 建议入口

### 4.3 节奏与分镜

- 一期“根据项目生成节奏规划”属于规则生成，不是大模型结果
- 当前节拍点可由规则生成，也已经预留音频上传与自动节拍识别能力
- 分镜生成时支持“是否适配节拍”开关
- 一期不支持策略性复用同一素材来凑时长
- 如果素材不够，按素材可用上限为准，不靠重复镜头硬补总时长
- 若业务上确实需要同一源素材多次使用，临时方案是人工先切分成多个素材，再分别录入系统

### 4.4 分镜排序

- 当前采用“两层排序”
- 第一层：按功能标签阶段排序
- 第二层：同一功能标签内按素材录入顺序排序

当前推荐排序思路：

1. 开头钩子
2. 地点建立 / 人物出场 / 过渡缓冲
3. 节奏刺点
4. 主高潮 / 慢高潮
5. 情绪收束 / 结尾收束

### 4.5 校验与交互

- 主题、节奏、分镜、导出页面都已经做过一轮工作流联动修复
- 重要页面已补充加载态、成功态和错误提示
- 分镜编辑页现在以“保存并返回列表”为主交互

## 5. 一期明确不做的内容

- 不做自动粗剪或自动成片
- 不做复杂策略性镜头复用引擎
- 不做完整多用户系统
- 不做深度音频算法引擎优化
- 不做最终级别的大模型智能编排排序

## 6. 一期技术形态

- 前端：Next.js
- 后端：FastAPI
- 数据库：SQLite
- 当前架构目标：一期页面层保持稳定，二期在服务层继续扩展

## 7. 二期已先行落地的孤立功能

以下能力已经先行实现，可直接作为二期开端继续扩展：

- 音频上传与自动节拍识别入口
- LLM 主题建议
- LLM 分镜建议
- LLM 导出建议
- 通用 LLM 配置项抽象，不绑定单一平台命名

## 8. 二期接手时要注意的点

- 文档已统一迁移到 `docs` 目录，优先以当前文档为准，不再回看旧讨论内容做口径判断
- 用户系统当前只是起步阶段，后续应转为后台数据库存储与后端校验
- 节奏、分镜、导出的“AI 建议”能力已打通入口，但算法和体验都还有迭代空间
- 目前还有一批技术债在 `phase2-master.md` 与 `phase2-checklist.md` 中统一管理
