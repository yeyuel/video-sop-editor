# 路线主轴与叙事骨架驱动的分镜编排技术方案

## 1. 背景

旅行短视频和攻略视频不能只按素材录入顺序或功能标签做简单排序。真实剪辑里同时存在两类约束：

- 路线主轴：旅行、攻略、Vlog 通常需要沿着真实路线或时间线推进，不能随意从后段地点跳回前段地点。
- 叙事骨架：短视频需要开头钩子、铺垫、反转、高潮、回收等节奏角色，不能把同一类“反转”连续堆在一起。

因此分镜生成需要从“素材排序”升级为“多约束编排”：先确定每个时间段的叙事角色，再在路线顺序约束下选择最合适的素材。

## 2. 设计目标

- 保持旅行路线连续性，正文镜头默认不倒退。
- 允许开头使用后段强素材做预告，但第二个镜头开始回到路线起点。
- 根据平台和内容类型生成不同叙事骨架。
- 在两次反转之间插入铺垫、发展、证明、缓冲等中间状态。
- 当素材不足时，宁可少用素材，也不要为了凑时长打乱路线。
- 所有自动选择结果都要能追溯，方便用户定位“为什么选了这个素材”。
- LLM 只作为增强建议，最终仍由规则校验素材真实存在、路线不乱跳、时长可控。

## 3. 核心概念

### 3.1 Route Axis

路线主轴表示项目的地点或路线顺序。当前来源优先级：

- 优先使用项目 `routeText` 拆分出的路线节点。
- 如果项目没有路线配置，则按素材录入时地点首次出现的顺序推断。
- 每个素材通过 `location` 匹配到一个 `routeIndex`。

路线匹配采用四层策略：

- 精确匹配：素材地点与路线节点完全一致。
- 规范化匹配：去掉空格、符号、大小写差异后匹配。
- 部分匹配：如“神仙居卧龙桥”可匹配路线节点“卧龙桥”。
- 模糊匹配：用于轻微错别字、英文拼写差异，当前使用 `SequenceMatcher`。

路线约束：

- `preview`：开头预告可跨路线选强画面，不推进正文路线位置。
- `forward`：只允许从当前路线节点向后选择。
- `strict`：攻略、章节型内容使用更严格的路线顺序。
- `flex_tail`：收尾优先选择后段或最后地点，不回退到早期地点。

### 3.2 Narrative Slot

叙事槽位表示某个时间段的功能角色，不等同于素材标签。当前由 `rhythmProfile.mode` 和 `attentionBeats` 生成。

抖音、快手类短视频示例：

| 槽位 | 作用 |
| --- | --- |
| `hook` | 开头 1-3 秒给强画面、强结果或强问题 |
| `setup` | 回到路线起点，建立地点和观看预期 |
| `turn_1` | 第一次注意力回拉，可用节奏刺点、反差、地点变化 |
| `develop_1` | 解释第一次反转后的状态，避免连续反转 |
| `turn_2` | 第二次信息升级或视觉变化 |
| `develop_2` | 承接第二次反转，推进到高潮前 |
| `climax` | 主高潮，优先强视觉、高情绪素材 |
| `payoff` | 结尾回收，形成记忆点或行动提示 |

B 站攻略类示例：

| 槽位 | 作用 |
| --- | --- |
| `hook` | 说明视频解决什么问题 |
| `chapter_setup` | 建立章节和路线位置 |
| `proof` | 用真实画面证明判断，如路况、体验、避坑 |
| `detail` | 补充细节和信息密度 |
| `summary` | 总结建议，方便暂停、截图、回看 |

## 4. 当前已落地算法

### 4.1 槽位生成

后端会根据节奏规划生成关键槽位：

- 如果存在 `attentionBeats`，优先使用节奏里的注意力点。
- 如果没有注意力点，则按平台模式生成默认骨架。
- 关键槽位之间会自动插入中间槽位，例如 `setup`、`develop_1`、`develop_2`、`climax_build`。

这样可以避免出现连续多个镜头都被标成“第一次反转”或“第二次反转”的粗粒度问题。

### 4.2 路线锚点分配

每个非预告槽位会先分配一个路线锚点 `targetRouteIndex`：

```text
bodySlots = 所有非 preview 槽位
routeCount = 路线节点数量

for slotIndex in bodySlots:
  targetRouteIndex = round(slotIndex / (bodySlotCount - 1) * (routeCount - 1))
```

收尾槽位会强制锚定到最后一个路线节点。

示例：3 个路线节点、6 个正文槽位时，锚点约为：

```text
0, 0, 1, 1, 2, 2
```

这样可以避免某个地点素材很多时，系统一直消耗该地点素材，导致后续地点迟迟不出现。

### 4.3 候选池选择

每个槽位选素材前会先构建候选池：

- `preview`：全片强画面候选，不受路线限制。
- `forward` / `strict`：从当前路线位置之后，选择最接近路线锚点的地点。
- `flex_tail`：优先选择路线锚点之后的素材，避免结尾倒回早期地点。
- 如果没有合法候选，停止补镜，不再回退到早期地点硬凑镜头。

### 4.4 素材评分

候选池内再按分数选素材。评分因素包括：

- 槽位角色匹配：如 `hook`、`turn_1`、`climax` 优先强画面和节奏刺点。
- 景别：大景、航拍、中景、特写在不同槽位中权重不同。
- 信息量：攻略、证明、总结类槽位优先高信息量素材。
- 功能标签：开头钩子、节奏刺点、主高潮、结尾收束等。
- 视觉和情绪标签：雪山、日落、航拍、人像、夜景、强反差等。
- 路线连续性：越接近当前合法路线位置越稳定。

### 4.5 剩余素材处理

当前策略是“路线优先于素材用尽”：

- 主骨架完成后，只补充当前路线位置之后的剩余素材。
- 已经推进到后段后，不再把早期地点的剩余素材追加到尾部。
- 如果素材不足以覆盖目标时长，由人工补充素材、调整时长或重新生成。

## 5. 可追溯输出

每条自动生成的分镜都会写入 `selectionTrace`，用于解释选择过程：

- 生成来源：规则生成、剩余素材补齐、LLM 生成等。
- 槽位信息：槽位 ID、叙事角色、路线策略。
- 候选池信息：路线锚点、实际候选路线节点、候选数量。
- 素材匹配信息：素材地点、匹配到的路线节点、匹配方式、匹配分数。
- 选择依据：景别、信息量、视觉强度、功能标签、视觉标签。

前端列表和编辑页可展示这些追溯信息，方便判断“为什么卧龙桥被提前”或“为什么某个地点没有被选中”。

## 6. LLM 增强边界

LLM 后续可以增强：

- 根据主题生成更自然的叙事槽位说明。
- 为每个候选素材生成选择理由。
- 生成字幕、口播、标题、封面建议。
- 在规则候选结果上做重排建议。

LLM 不允许绕过：

- 素材 ID 必须真实存在。
- 正文路线不能倒退。
- 分镜总时长必须受目标时长约束。
- 不得自动复用同一素材，除非用户人工切分成多个素材条目。

## 7. 当前验收标准

- 开头钩子可以使用后段强素材，但正文回到路线起点。
- 正文分镜路线序号整体不倒退。
- 中间不再连续堆叠同一个反转角色。
- 多地点项目中，槽位会按路线锚点逐步推进到后续地点。
- 后段素材被选中后，不再把早期地点素材追加到结尾。
- 分镜列表可以查看每条镜头的算法追溯信息。

## 8. 已完成升级

### 8.1 Beam Search 多候选时间线

当前分镜选材已经从单路径贪心升级为小宽度 Beam Search：

- 每个槽位不只保留一个局部最优素材，而是保留多个候选时间线。
- 状态包含已选素材、剩余素材、当前路线位置和累计分数。
- 分数综合槽位匹配、路线锚点、画面强度、功能标签和路线连续性。
- 最终选择累计分数最高、槽位覆盖更完整、路线推进更合理的时间线。

这能减少“前一个槽位抢走后一个关键槽位素材”的问题。例如：

- `rhythm_hit` 更倾向留给 `turn_1` / `turn_2` / `climax`。
- `ending` 更倾向留给 `payoff` / `summary`。
- `opening_hook` 且位于后段的强画面更适合做开头预告。

### 8.2 素材复用桥段策略

开启素材复用后，系统只复用素材本身，不再复用第一轮的叙事角色，也不再把复用片段简单追加到时间线末尾。

错误方式：

```text
hook -> setup -> turn_1 -> develop_1 -> climax
-> hook -> setup -> turn_1 -> develop_1 -> climax
```

当前方式：

```text
hook -> bridge -> setup -> bridge -> turn_1 -> bridge -> develop_1
-> bridge -> turn_2 -> bridge -> climax -> bridge -> payoff
```

实现原则：

- 先生成一条不复用的主叙事骨架，确保 `hook / turn_1 / turn_2 / climax / payoff` 顺序固定。
- 估算主骨架时长，如果低于目标时长，再计算需要补足的时长。
- 复用素材只作为 `bridge` 插入到主骨架节点之间，不作为第二轮主叙事。
- 插入桥段时必须给后续主骨架锚点预留时长，不能因为前段补时导致 `climax / payoff / ending` 被挤出最终时间线。
- 每个主骨架间隙最多插入少量桥段，每条素材也有复用次数上限；素材不足时宁可提示补素材或人工切分，不为了凑时长反复刷同一个缓冲镜头。
- 优先复用低风险素材，如 `supporting`、`transition_buffer`、`setup`、`develop` 类素材。
- 避免优先复用 `opening_hook`、`rhythm_hit`、`main_climax`、`ending` 等结构性素材。
- 每条复用桥段都会在 `selectionTrace` 中标记 `[reuse-bridge]`，便于 UI 追溯。

不同模式下的复用补时角色：

- 抖音 / 快手：复用段降级为 `supporting`。
- 小红书：复用段降级为 `experience`。
- B 站攻略：复用段降级为 `detail`。
- B 站 Vlog：复用段降级为 `afterglow`。

这样可以避免第二轮又出现“开头钩子、第一次反转、第二次反转、高潮”的结构性重复，同时让多次“射门点”之间有更多过渡素材承接。

### 8.3 平台化短时长骨架

小于 20 秒的视频不再统一使用 `hook -> push -> climax -> ending`，而是按平台生态生成不同骨架：

- 抖音 / 快手：`hook -> turn_1 -> turn_2 -> climax -> payoff`，保留“射门集锦式”连续注意力回拉。
- 小红书：`hook -> visual_seed -> info_value -> save_cta`，突出种草、信息价值和收藏动作。
- B 站攻略：`hook -> chapter -> proof -> summary`，优先章节感和可验证信息。
- B 站 Vlog：`hook -> immersion -> emotional_climax -> aftertaste`，保留沉浸、情绪和余味。

## 9. 后续升级方向

- LLM Route Match：对复杂地点名、别名、景区子点位做智能匹配，并把 LLM 判断理由写入追溯信息。
- 节拍对齐优化：把剪映式节拍识别结果纳入候选镜头时长决策，减少自动节拍和实际卡点的偏差。
- 自动口播：根据字幕或预设文案生成旁白音频，并反向影响镜头停留时长。
- Beam Search 权重可配置：后续可以在 UI 中暴露“路线优先 / 戏剧张力优先 / 信息密度优先”等模式。
- 分镜质量评分：生成后给出路线连续性、节奏密度、钩子强度、结尾回收等评分，便于用户快速判断版本质量。
