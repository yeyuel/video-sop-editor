# 旅行短视频剪辑导演助手 SOP v1.1

## 1. 文档目标

本文档定义一期 Web App 的标准作业流程，用于指导产品、开发、测试和运营协作。

一期核心目标：

- 输入项目基础信息和素材信息
- 生成主题方向
- 先做节奏规划，再输出可执行分镜
- 在导出阶段补齐标题、标签和文案，并写入导出脚本

## 2. 一期范围

包含：

- 项目管理
- 素材录入与维护
- 主题生成与选择
- 节奏规划
- 分镜生成
- 导出预览

不包含：

- 自动成片
- 真实歌曲音频节拍识别
- LLM 自动生成发布文案

## 3. 参与角色

- 用户：录入项目和素材，选择主题，调整节奏与分镜，填写导出信息
- 系统：生成主题、节奏规划、分镜时间线和导出脚本
- 剪辑师：根据导出的时间线脚本完成剪辑

## 4. 标准流程

## 4.1 新建项目

输入：

- 项目名称
- 目的地
- 发布平台
- 目标时长
- 视频类型
- 风格偏好
- 风格补充说明
- 路线
- 素材根目录

处理规则：

- 新建项目默认状态为 `draft`
- 状态不在创建时手动选择
- 项目级保存素材根目录

输出：

- 项目 ID
- 项目基础档案

验收：

- 项目可创建、编辑、删除
- 首页可查看历史项目列表

## 4.2 素材录入

输入：

- 地点
- 画面内容
- 素材相对路径
- 视频类型
- 镜头类型
- 信息量
- 建议时长
- 情绪标签
- 视觉标签
- 功能标签

处理规则：

- 素材 ID 自动生成
- 素材相对路径保存在素材级
- 素材根目录保存在项目级
- 功能标签支持推荐项和自定义项
- 镜头类型必须支持“中景含人物/主体”

镜头类型说明：

- 大景
- 中景
- 中景含人物/主体
- 特写
- 人物
- 动物
- 转场

输出：

- 结构化素材列表

验收：

- 支持素材新增、编辑、删除
- 素材列表可直接回首页或项目页

## 4.3 主题生成与确认

前置条件：

- 当前项目下至少存在一条素材

处理流程：

1. 用户点击生成主题
2. 系统输出 3 个主题方向
3. 用户选择当前主题
4. 当前主题保存到项目对象

输出字段：

- 标题
- 摘要
- 核心情绪
- 预期节奏
- 平台理由
- 是否选中

验收：

- 主题可重新生成
- 当前主题只能有一个

## 4.4 节奏规划

前置条件：

- 已选择当前主题

处理流程：

1. 用户点击生成节奏规划
2. 系统输出节拍模式、节拍点和节奏说明
3. 用户可人工修改
4. 保存为项目当前节奏规划

输出字段：

- BGM 风格
- 参考曲目
- 节拍模式
- 节拍点
- 节奏说明
- 暗场建议点
- 照片动效建议

关键规则：

- 分镜依赖节拍点，因此顺序必须是“节奏 → 分镜”
- 当前节拍点为规则生成，不是从真实歌曲做音频识别
- 后续可替换成类似剪映的真实 beat 点识别
- 当前不支持上传音频文件，也不做波形、BPM、强拍弱拍或 downbeat 自动分析
- 当前“生成节奏规划”仅基于项目目标时长、视频类型和已选主题做规则推导，再由用户人工修正

二期预留要求：

- 支持上传音频文件作为节奏参考源
- 系统自动提取 BPM、节拍点、强拍/弱拍和段落变化
- 自动识别结果写回当前节奏规划，但仍保留手工修改入口
- 当音频识别失败或未上传音频时，回退到当前规则生成模式

验收：

- 用户可生成并保存节奏规划
- 保存后 workflow 解锁分镜阶段

## 4.5 分镜生成

前置条件：

- 已有当前主题
- 已有节拍点

处理流程：

1. 用户点击生成分镜
2. 系统根据素材、目标时长、当前主题、节拍点生成时间线
3. 用户可人工编辑分镜内容
4. 用户保存分镜

输出字段：

- `startTime`
- `endTime`
- `assetId`
- `shotDescription`
- `function`
- `rhythm`
- `beatMode`
- `beatPoints`
- `subtitle`

校验项：

- 是否全部绑定真实素材
- 是否通过地点连续性检查
- 是否通过节拍对齐检查
- 当前总时长是多少

关键规则：

- 不允许产生没有 `assetId` 的分镜
- 分镜时长优先贴合节拍点
- 分镜可人工微调后再保存

验收：

- 能生成可编辑的时间线
- 校验结果可见
- 分镜列表页只展示关键信息，不承载整表编辑
- 单条分镜进入独立编辑页，保存后自动返回列表页

分镜列表维护规则：

- 当前列表页支持查看、进入编辑、删除分镜
- 新增分镜采用“在当前镜头后插一条”的方式
- 调整顺序采用“上移 / 下移”方式，不要求一期支持拖拽排序
- 新增或调整顺序后，需要重新计算后续镜头的开始时间和结束时间
- 时间重排时优先保持每条镜头自身时长不变，再按新顺序重铺时间线

新增分镜规则：

- 默认插入到当前镜头之后
- 默认 `start_time` 等于前一镜头 `end_time`
- 默认 `end_time` 按 1 秒或项目常用镜头时长预填
- 默认继承当前项目节拍模式，节拍点可人工补充或调整

顺序调整规则：

- 当前实现中，分镜展示顺序以时间线顺序为准
- 上移 / 下移的本质是调整镜头顺序后重算时间轴
- 一期暂不引入独立 `position` 字段，也不做可视化拖拽

## 4.6 导出

前置条件：

- 已存在分镜时间线

处理流程：

1. 用户填写标题、短标题、标签、文案、封面建议
2. 用户保存导出信息
3. 用户按需预览 Markdown / JSON / YAML

导出规则：

- 导出内容必须包含：
  - 项目基础信息
  - 当前主题
  - 节奏规划
  - 分镜时间线
  - 标题
  - 标签
  - 文案
- 当前标题、标签、文案由人工填写
- 二期再加入 LLM 自动建议

验收：

- 三种格式均可生成预览
- 导出内容中能看到时间线脚本和导出文案

## 5. Workflow 解锁规则

1. 新建完成后解锁录入
2. 有素材后解锁主题
3. 有选中主题后解锁节奏
4. 有节拍点后解锁分镜
5. 有分镜后解锁导出

## 6. UI 规则

- workflow stepper 是主要阶段导航
- stepper 已解锁节点可点击跳转
- 当前高亮必须和页面功能一致
- 项目页不再额外保留重复的 workflow 导航卡片

## 7. 一期验收标准

- 项目 CRUD 可用
- 素材 CRUD 可用
- 主题生成与选择可用
- 节奏规划生成与保存可用
- 分镜生成与保存可用
- 导出信息保存与多格式预览可用
- workflow 每个 step 都能按顺序解锁并运行

## 8. 节奏字段实施边界

### 8.1 一期规则生成字段

一期节奏规划中，以下字段继续采用规则生成或模板占位方式：

- `beatMode`
- `beatPoints`
- `darkCutSuggestions`
- `photoMotionSuggestions`
- `selectedTrackName`
- `rhythmNotes`

执行说明：

- `beatMode` 按视频类型做规则映射。
- `beatPoints` 按目标时长和固定时间间隔生成。
- `darkCutSuggestions` 按结构比例点生成。
- `selectedTrackName` 使用 demo 占位命名。
- `photoMotionSuggestions` 按是否存在照片素材返回模板建议。
- `rhythmNotes` 基于主题、素材和项目字段拼接生成。

### 8.2 当前不应误判为真实分析的字段

- 当前 `beatPoints` 不是从真实歌曲波形中提取。
- 当前 `beatMode` 不是根据 BPM、downbeat 或强弱拍自动分析。
- 当前 `darkCutSuggestions` 不是根据音乐段落识别生成。
- 当前 `selectedTrackName` 不是实际曲库或上传音轨名称。
- 当前 `bgmStyle` 和 `rhythmNotes` 不是 LLM 实时生成结果。

### 8.3 二期升级优先级

1. `beatPoints`
2. `selectedTrackName`
3. `darkCutSuggestions`
4. `beatMode`
5. `rhythmNotes`
6. `bgmStyle`
7. `photoMotionSuggestions`

### 8.4 二期替换目标

- 支持上传音频文件作为节奏参考源。
- 自动提取真实 beat points。
- 自动识别 BPM、强拍/弱拍和段落变化。
- 将 `selectedTrackName` 替换为真实音轨信息。
- 将 `rhythmNotes` 和 `bgmStyle` 升级为 LLM 辅助建议。
- 当真实音频识别失败时，回退到一期规则生成模式。
## 9. 二期 LLM 接入 SOP 约束

### 9.1 接入前确认

每新增一个 LLM provider，先确认：

- 是否提供官方 API Key
- 是否提供官方 OAuth / OIDC
- 是否提供 Device Code Flow
- 是否允许第三方应用正式授权

若以上都不提供，则该 provider 不进入正式接入流程。

### 9.2 标准接入顺序

1. 先登记 provider 元数据。
2. 确认认证模式。
3. 配置模型、地址、scope 和回调信息。
4. 完成连通性验证。
5. 再开放给主题、分镜、导出业务使用。

### 9.3 安全要求

- API Key 只保存在后端。
- OAuth / Device Code token 只保存在后端。
- 前端不缓存长期敏感凭证。
- 失效 token 必须可撤销、可刷新、可标记异常。

### 9.4 禁止事项

- 禁止手工复制网页 cookie 作为系统凭证。
- 禁止抓取用户浏览器 session。
- 禁止依赖非官方网页登录态模拟 LLM 能力调用。
