# 旅行短视频剪辑导演助手 PRD v1.1

## 1. 产品目标

一期目标是把旅行视频剪辑前期最容易混乱的部分结构化：

- 管理项目基础信息
- 录入和维护素材清单
- 基于素材生成主题方向
- 先做节奏规划，再生成分镜时间线
- 在导出阶段整理标题、标签、文案，并把这些信息写入最终导出脚本

一期不做：

- 自动成片
- 自动识别真实音频节拍文件
- LLM 自动生成标题/标签/文案

说明：

- 如果 PRD 与现有代码逻辑冲突，以当前已实现逻辑为准。
- 二期再在现有接口上补充视频分析、真实节拍识别和 LLM 建议生成。

## 2. 目标用户

- 旅行内容创作者
- 剪辑师
- 运营同学

## 3. 核心价值

- 把“素材很多、不会组织”的问题，转成可执行 workflow。
- 保证每个分镜都绑定真实素材，降低臆造风险。
- 让节奏规划先行，避免分镜先写完后才发现卡点对不上。

## 4. Workflow

固定顺序：

1. 新建
2. 录入
3. 主题
4. 节奏
5. 分镜
6. 导出

阶段依赖：

- 没有项目，不能进入素材录入
- 没有素材，不能进入主题
- 没有选定主题，不能进入节奏
- 没有节拍点，不能进入分镜
- 没有分镜，不能进入导出

## 5. 功能需求

## 5.1 项目管理

首页展示历史项目列表，支持：

- 新建项目
- 进入项目
- 进入素材列表
- 删除项目

项目字段：

- 项目名称
- 目的地
- 发布平台
- 目标时长
- 视频类型
- 风格偏好
- 风格补充说明
- 路线
- 素材根目录
- 状态

规则：

- 新建项目默认状态为 `draft`
- 项目级保存素材根目录
- 当前不允许在新建项目时手动选择状态

## 5.2 素材 CRUD

素材列表页支持：

- 查看素材列表
- 新增素材
- 编辑素材
- 删除素材

素材字段：

- 系统自动生成素材 ID
- 地点
- 画面内容
- 素材相对路径
- 视频类型
- 镜头类型
- 信息量
- 建议时长
- 情绪标签
- 视觉标签
- 功能标签

镜头类型枚举：

- 大景
- 中景
- 中景含人物/主体
- 特写
- 人物
- 动物
- 转场

说明：

- “中景含人物/主体”表示同时包含景别信息和人物/主体语义
- 功能标签支持推荐选项 + 自定义输入

## 5.3 主题生成与选择

功能：

- 基于项目和素材生成 3 个主题方向
- 用户选择一个当前主题

输出字段：

- 标题
- 摘要
- 核心情绪
- 预期节奏
- 平台理由
- 是否当前选中

规则：

- 当前主题保存在项目上
- 后续节奏和分镜都依赖当前主题

## 5.4 节奏规划

功能：

- 生成节奏方案
- 人工编辑后保存

字段：

- BGM 风格
- 参考曲目
- 节拍模式
- 节拍点
- 节奏说明
- 暗场建议点
- 照片动效建议

规则：

- 分镜生成依赖节拍点
- 当前实现是按规则生成 beat points，不是从真实歌曲文件里做音频识别
- 二期可以替换为真实歌曲节拍识别能力
- 当前不支持上传音频文件，也不做波形、BPM、强拍弱拍或 downbeat 自动分析
- 当前“生成节奏规划”按钮的行为是：
  - 根据视频类型推荐节拍模式
  - 根据目标时长按固定时间间隔生成 beat points
  - 允许用户继续手工编辑节拍点、暗场建议点和节奏说明

二期目标：

- 支持上传音频文件作为节奏分析输入
- 系统自动识别 BPM、beat points、强拍/弱拍和段落变化
- 自动识别结果写入当前 rhythm plan，并允许用户人工修正
- 当没有上传音频或识别失败时，回退到一期的规则生成方案

## 5.5 分镜生成

功能：

- 基于当前主题、素材、目标时长、节拍点生成时间线
- 生成后允许人工微调并保存

字段：

- startTime
- endTime
- assetId
- shotDescription
- function
- rhythm
- beatMode
- beatPoints
- subtitle

规则：

- 所有分镜必须绑定真实素材 ID
- 分镜生成顺序必须在节奏规划之后
- 时间长度优先贴合节拍点
- 校验结果要返回：
  - 是否全部绑定素材
  - 是否通过地点连续性检查
  - 是否通过节拍对齐检查
  - 当前总时长

列表与编辑交互：

- 分镜列表页只显示摘要信息，不做整表表单编辑
- 单条分镜进入独立编辑页处理详细字段
- 编辑页保存后自动返回分镜列表，适合快速逐条修正
- 列表页保留的动作包括：
  - 编辑
  - 删除
  - 在当前镜头后插一条
  - 上移 / 下移

一期最小改动方案：

- 新增分镜采用“后插一条”，不做任意位置拖拽插入
- 调整顺序采用“上移 / 下移”，不做自由拖拽排序
- 顺序变化后，系统按新顺序重算 `startTime / endTime`
- 重算时间轴时，优先保持每条分镜自身时长不变
- 当前阶段仍沿用时间线顺序表达列表顺序，不单独增加 `position` 字段

## 5.6 导出

导出阶段当前做两件事：

1. 人工填写导出信息
2. 预览导出脚本

导出信息字段：

- 标题
- 短标题
- 标签
- 文案
- 封面建议

预览格式：

- Markdown
- JSON
- YAML

规则：

- 导出内容必须带上标题、标签、文案和分镜时间线
- 当前为人工填写
- 二期再补 LLM 建议生成，但字段结构保持不变

## 6. 页面要求

## 6.1 首页

- 历史项目列表
- 右上角新建项目按钮
- 删除项目二次确认

## 6.2 项目页

- 上方展示项目信息
- 中间展示可点击的 workflow stepper
- 下方展示项目编辑和数据状态

说明：

- workflow stepper 本身承担导航作用
- 不再额外做一套重复的 workflow 导航卡片

## 6.3 各阶段页面

- 保留 breadcrumb
- 保留 workflow stepper
- 当前 step 高亮必须与实际页面一致
- stepper 中已解锁阶段可点击跳转

## 6.4 分镜列表页

- 顶部保留“根据节拍生成分镜”入口
- 列表每条分镜仅显示：
  - 时间范围
  - 素材 ID
  - 镜头描述
  - 功能标签
- 列表页不出现“保存整表分镜”按钮
- 所有详细编辑动作下沉到单镜头编辑页

## 6.5 单镜头编辑页

- 用于快速修正单条分镜
- 保存后自动返回分镜列表
- 允许编辑：
  - 开始时间
  - 结束时间
  - 素材 ID
  - 功能标签
  - 镜头描述
  - 节奏说明
  - 节拍模式
  - 字幕建议
  - 当前镜头节拍点

## 7. 当前已实现范围

- 项目 CRUD
- 素材 CRUD
- 主题生成与选择
- 节奏规划生成与保存
- 分镜生成与保存
- 导出信息保存
- Markdown / JSON / YAML 预览导出

## 8. 二期扩展方向

- 真实歌曲节拍识别
- 音频文件上传与节拍自动识别
- 节拍识别失败时回退到规则生成
- 视频内容自动分析
- 素材自动标签补全
- LLM 自动生成标题、标签、文案
- 自动粗剪
