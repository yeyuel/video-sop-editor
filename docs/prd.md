# 旅行短视频剪辑导演助手 PRD v1.2

## 1. 产品目标

一期目标是把旅行短视频剪辑前期最容易混乱的部分结构化，帮助用户稳定完成以下工作：

- 管理项目基础信息
- 录入和维护素材清单
- 基于素材生成主题方向
- 先做节奏规划，再生成分镜时间线
- 在导出阶段整理标题、标签和文案

一期不做：

- 自动成片
- 自动识别真实音频节拍文件
- LLM 自动生成标题、标签和文案
- 自动策略性复用同一素材

说明：

- 如果 PRD 与当前代码逻辑冲突，以当前已实现逻辑为准。
- 二期再在现有接口上补充视频分析、真实节拍识别和 LLM 建议生成。

## 2. 目标用户

- 旅行内容创作者
- 剪辑师
- 运营同学

## 3. 核心价值

- 把“素材很多、不会组织”的问题，转成可执行 workflow。
- 保证每个分镜都绑定真实素材，降低臆造风险。
- 让节奏规划先行，避免分镜写完后才发现卡点对不上。

## 4. Workflow

固定顺序：

1. 新建
2. 录入
3. 主题
4. 节奏
5. 分镜
6. 导出

阶段依赖：

- 没有项目，不能进入素材录入
- 没有素材，不能进入主题
- 没有选定主题，不能进入节奏
- 没有节拍点，仍可生成分镜，但若关闭节拍适配则按素材建议时长生成
- 没有分镜，不能进入导出

## 5. 功能需求

## 5.1 项目管理

首页展示历史项目列表，支持：

- 新建项目
- 进入项目
- 进入素材列表
- 删除项目

项目字段：

- 项目名称
- 目的地
- 发布平台
- 目标时长
- 视频类型
- 风格偏好
- 风格补充说明
- 路线
- 素材根目录
- 状态

规则：

- 新建项目默认状态为 `draft`
- 项目级保存素材根目录
- 当前不允许在新建项目时手动选择状态

## 5.2 素材 CRUD

素材列表页支持：

- 查看素材列表
- 新增素材
- 编辑素材
- 删除素材

素材字段：

- 系统自动生成素材 ID
- 地点
- 画面内容
- 素材相对路径
- 视频类型
- 镜头类型
- 信息量
- 建议时长
- 情绪标签
- 视觉标签
- 功能标签

镜头类型枚举：

- 大景
- 中景
- 中景含人物/主体
- 特写
- 人物
- 动物
- 转场

说明：

- “中景含人物/主体”表示同时包含景别信息和人物/主体语义。
- 功能标签支持推荐选项 + 自定义输入。
- 如果一个原始视频中包含多个可独立使用的镜头，建议用户先人工切分，再分别作为多个素材录入系统。

## 5.3 主题生成与选择

功能：

- 基于项目和素材生成 3 个主题方向
- 用户选择一个当前主题

输出字段：

- 标题
- 摘要
- 核心情绪
- 预期节奏
- 平台理由
- 是否当前选中

规则：

- 当前主题保存在项目上
- 后续节奏和分镜都依赖当前主题

## 5.4 节奏规划

功能：

- 生成节奏方案
- 人工编辑后保存

字段：

- BGM 风格
- 参考曲目
- 节拍模式
- 节拍点
- 节奏说明
- 暗场建议点
- 照片动效建议

规则：

- 分镜生成可选择是否适配节拍。
- 支持规则生成 beat points，也支持上传音频文件做基础节拍识别。
- 音频上传支持 WAV、MP3、M4A、AAC、OGG、MGG、 FLAC、WMA；非 WAV 格式需本机安装 ffmpeg 转码。
- 识别结果写入 `analysisSource`（`rule` / `audio_upload` / `manual`），失败时可回退规则生成。
- 音频上传后优先使用 librosa 识别真实节拍；不可用时回退能量起音检测。
- 支持剪映对标踩点模式：踩节拍1（粗密度）、踩节拍2（细密度）、强弱拍；切换模式基于 `rawBeatPoints` 重采点。
- 规则生成路径仍保留，作为无音频或识别失败时的兜底。

## 5.5 分镜生成

功能：

- 基于当前主题、素材、目标时长、节拍点生成时间线
- 生成后允许人工微调并保存

字段：

- startTime
- endTime
- assetId
- shotDescription
- function
- rhythm
- beatMode
- beatPoints
- subtitle

一期生成规则：

- 所有分镜必须绑定真实素材 ID
- 一期默认不自动重复使用同一素材
- 分镜生成顺序必须在节奏规划之后
- 分镜生成时，素材采用两层排序：
- 第一层按主叙事阶段排序：`opening_hook` → `supporting` → `slow_climax` → `main_climax` → `ending`
- 第二层按段内修饰排序：普通镜头优先，其次是 `rhythm_hit`，最后是 `transition_buffer`
- 同一层级内，按素材录入顺序排列
- 未设置主叙事标签的素材，默认归入 `supporting` 段
- `rhythm_hit` 和 `transition_buffer` 不再作为全局章节标签使用，而是附着在当前叙事段内参与微调
- 若开启“适配节拍”，镜头时长优先贴合节拍点
- 若关闭“适配节拍”，镜头按素材建议时长顺排
- 当素材全部使用完仍不足目标时长时，系统停止生成，并以当前可生成的最大时长为准
- 系统需要明确提示“素材不足”，并建议用户补充素材，或先将长素材切分后分别录入

说明：

- 如果确实需要同一原始视频里的多个镜头，请优先通过“人工先切分，再作为多个素材录入”的方式解决，而不是在一期里自动复用同一素材。

校验结果需要返回：

- 是否全部绑定素材
- 是否通过地点连续性检查
- 是否启用节拍适配
- 若启用节拍适配，是否通过节拍对齐检查
- 当前总时长
- 是否达到目标时长
- 结果说明文案

列表与编辑交互：

- 分镜列表页只显示摘要信息，不做整表单编辑
- 单条分镜进入独立编辑页处理详细字段
- 编辑页保存后自动返回分镜列表
- 列表页保留的动作包括：
  - 编辑
  - 删除
  - 在当前镜头后插一条
  - 上移 / 下移

## 8.1 节奏字段升级边界

一期继续保留规则生成的字段：

- `beatMode`
- `beatPoints`
- `darkCutSuggestions`
- `photoMotionSuggestions`
- `selectedTrackName`
- `rhythmNotes`

当前实现说明（Sprint 2 后）：

- `beatPoints`：音频上传时由 librosa / 能量检测识别；规则生成时由目标时长 + 剪映踩点语义估算；识别失败回退规则生成。
- `beatMode`：音频上传时按 BPM 与节拍间隔推荐（快歌 → 踩节拍2 细密度，中速 → 踩节拍1 粗密度，慢歌 → 强弱拍）；规则生成时按视频类型映射；语义对齐剪映（1 粗 / 2 细）。
- `rawBeatPoints`：缓存完整识别节拍，切换 `beatMode` 时不需重新上传音频。
- `selectedTrackName`：音频上传时部分使用文件名；规则生成仍为 demo 占位，待 Sprint 4 统一。
- `darkCutSuggestions`：仍由固定结构点（25%/50%/75%）推导，待 Sprint 4 升级为音频段落识别。
- `bgmStyle` 和 `rhythmNotes`：仍基于主题、素材和项目字段做模板拼接，待 Sprint 4 LLM 化。

## 8.2 二期 LLM 接入要求

二期中，LLM 接入必须设计为“厂商无关配置 + 多认证模式兼容”，不能把产品能力绑定为单一 OpenAI 命名体系。

支持模式：

1. `api_key`
2. `oauth`
3. `device_code`

业务要求：

- 主题建议、分镜建议、导出建议统一走 LLM 网关。
- 上层业务不直接区分 ChatGPT、Kimi、GLM、DeepSeek 等厂商。
- 当前厂商如果仅提供 OpenAI-compatible API，则按 `api_key` 模式接入。
- 当前厂商如果提供官方 OAuth / OIDC，则支持浏览器跳转授权。
- 当前厂商如果提供 Device Code，则支持设备授权流程。

明确边界：

- 只有网页账号登录但没有官方开放协议的产品，不纳入正式接入范围。
- 不允许通过抓取 cookie、session、网页存储令牌等非官方方式接入。
