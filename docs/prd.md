# 旅行短视频剪辑导演助手 PRD v1

## 1. 文档信息

- 产品名称：旅行短视频剪辑导演助手
- 版本：v1.0
- 阶段：一期 MVP
- 文档用途：指导产品设计、研发实现、测试验收和后续迭代

## 2. 产品背景

旅行短视频创作者通常拥有大量素材，但缺少稳定的方法完成以下工作：

- 整理素材
- 识别叙事主线
- 安排镜头节奏
- 组织字幕和音乐
- 生成发布文案

目前常见工具偏向剪辑执行层，缺少“导演层”的结构化辅助能力。本产品的切入点不是直接替代剪辑师，而是先帮助用户完成从素材到分镜脚本的决策工作。

## 3. 产品目标

一期目标：

- 基于素材脑图和素材卡片生成结构化剪辑方案。
- 保证分镜只引用真实素材，不创造不存在的画面。
- 让用户能直接把输出结果拿去人工剪辑和发布。
- 在用户已选定 BGM 的场景下，支持按节拍点辅助生成镜头秒数。

非目标：

- 一期不做自动生成高质量成片。
- 一期不强求全自动视频识别。
- 一期不做复杂团队协同和权限系统。

## 4. 用户与场景

目标用户：

- 旅行短视频创作者
- 个人 Vlogger
- 小团队剪辑师
- 内容运营人员

典型场景：

- 旅行混剪
- 情绪片
- 城市短片
- 路线纪实片
- 攻略型短视频

核心痛点：

- 素材很多，不知道如何筛选和排序
- 地点、路线、情绪经常剪乱
- 不会设计开头钩子和中段爆点
- 歌曲节奏和分镜秒数容易冲突
- 发布文案产出效率低

## 5. 产品定位

- 产品定义：旅行短视频剪辑导演助手
- 核心输出：可执行剪辑脚本
- 差异化方向：把素材信息量、镜头功能、平台逻辑和节奏设计结构化

## 6. 成功指标

一期成功标准：

- 输入素材脑图后，系统能生成不创造素材、可直接执行的分镜脚本。
- 至少 3 个真实项目验证通过。
- 用户能够导出 Markdown、JSON、YAML、PDF 并用于人工剪辑。
- 在已选 BGM 的测试项目里，分镜秒数与节拍卡点不出现明显冲突。

建议监控指标：

- 项目创建完成率
- 素材录入完成率
- 主题采纳率
- 分镜导出率
- 分镜人工修改率
- 臆造素材错误率
- 节拍卡点人工调整率

## 7. 功能范围

## 7.1 核心功能

1. 项目创建
2. 素材录入
3. 素材脑图整理
4. 叙事主题生成
5. 分镜生成
6. 音乐节奏方案
7. 发布方案
8. 导出

## 7.2 一期范围内

- 支持平台、时长、风格、路线等项目配置
- 支持手动素材卡和素材脑图输入
- 支持素材分组和基础编辑
- 支持基于素材生成多个叙事方向
- 支持生成逐秒分镜时间线
- 支持在已选 BGM 场景下输出节拍点建议
- 支持生成发布文案
- 支持 Markdown、JSON、YAML、PDF 导出

## 7.3 一期范围外

- 自动识别整段视频内容
- 自动生成完整成片
- 剪映 GUI 自动化
- 复杂素材库存储系统
- 多人实时协作

## 8. 用户流程

1. 用户创建项目。
2. 用户选择平台、时长、风格和路线。
3. 用户录入素材脑图和素材卡片。
4. 系统整理结构化素材表。
5. 系统生成 3 至 5 个主题方向。
6. 用户选择主题方向。
7. 用户可选填 BGM 或节拍偏好。
8. 系统生成分镜时间线。
9. 用户手动替换或调整局部镜头。
10. 系统生成音乐节奏建议和节拍点。
11. 系统生成标题、描述、标签、封面建议。
12. 用户导出 Markdown、JSON、YAML、PDF。

## 9. 页面需求

## 9.1 项目列表页

目标：

- 展示所有项目和处理状态

核心元素：

- 项目名
- 平台
- 视频类型
- 最后更新时间
- 状态标签

核心操作：

- 新建项目
- 搜索项目
- 进入详情

## 9.2 创建项目页

输入项：

- 项目名
- 目的地
- 发布平台
- 目标时长
- 视频类型
- 风格偏好
- 路线
- 素材录入方式

校验规则：

- 项目名必填
- 平台必填
- 时长必须为合法区间值

## 9.3 素材录入页

输入能力：

- 粘贴素材脑图
- 手动创建素材卡片
- 上传截图
- 粘贴文件名列表

输出能力：

- 结构化素材表
- 素材分组结果

关键字段要求：

- `shotType` 必须包含 `中景`
- `中景` 用于表示兼顾环境和人物或主体关系的镜头

## 9.4 素材脑图页

能力要求：

- 按地点、场景、镜头功能分组
- 支持拖拽排序
- 支持编辑节点

## 9.5 主题选择页

能力要求：

- 生成多个叙事方向
- 展示每个方向的定位、节奏和适配平台理由
- 支持用户选定一个主主题

## 9.6 分镜生成页

能力要求：

- 输出逐秒时间线
- 支持手动替换素材
- 支持修改时长和字幕
- 标识每个镜头的功能位和节奏位
- 已选 BGM 时展示节拍点和卡点结果

## 9.7 音乐节奏页

能力要求：

- 输出 BGM 风格建议
- 输出快慢节奏和暗场位置
- 输出照片慢推和转场建议
- 支持展示 `节拍 1`、`节拍 2` 或强弱拍锚点

## 9.8 发布方案页

能力要求：

- 生成标题
- 生成短标题
- 生成描述和标签
- 生成封面建议

## 9.9 导出页

能力要求：

- 导出 Markdown
- 导出 JSON
- 导出 YAML
- 导出 PDF

## 11. 关键规则

- 所有分镜必须引用真实 `assetId`。
- 没有 `assetId` 的内容不得进入最终分镜。
- 分镜生成必须遵守路线和地点归属连续性。
- 镜头时长分配要参考信息量和情绪强度。
- 平台要影响脚本节奏与发布文案风格。
- 用户已选择 BGM 时，镜头时长应优先贴合节拍点。

## 12. 技术方案建议

- 前端：Next.js + React + Tailwind + shadcn/ui
- 后端：Next.js API Routes 或 FastAPI
- 数据库：SQLite 起步，后续可升级 PostgreSQL
- 文件存储：本地文件夹起步，后续可切换 S3/R2
- 模型层：OpenAI API 或可替换模型接口
- 导出：Markdown / JSON / YAML / PDF

## 14. 风险与控制

| 风险 | 表现 | 控制策略 |
| --- | --- | --- |
| 模型臆造素材 | 生成不存在的镜头 | 分镜必须强制引用 `assetId` |
| 地点归属混乱 | 不同地点镜头被错误串联 | 素材表必须保留 `location` 并在生成时校验 |
| 过早自动化识别 | 识别不准且成本高 | 一期优先人工确认和半自动整理 |
| 自动剪辑质感不足 | 粗剪机械 | 二期目标只定义为 60 分粗剪 |
| 节拍与分镜冲突 | 卡点不准，人工返工高 | 输出 `beatMode` 与 `beatPoints`，并加入校验 |
