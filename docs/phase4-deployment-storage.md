# P4 部署与存储评估

## 1. 结论

首发继续采用单机部署：`Next.js + 单进程 FastAPI + SQLite WAL + 本地 STORAGE_DIR`。

该方案适合当前本人使用和少量朋友试用，目标容量为最多约 10 个同时在线用户。暂不引入 Postgres、Redis、Celery 或对象存储，避免把维护成本提前转化为业务成本。

## 2. 10 用户下 SQLite 边界

当前主要请求是页面读取、短事务 CRUD、LLM 网络等待和少量音频分析。SQLite 的问题不在读取，而在同一时刻只能有一个写事务。

本阶段启用：

- `journal_mode=WAL`：读请求不再被普通写事务整体阻塞。
- `busy_timeout=5000ms`：短暂写锁冲突先等待，不立即向用户报错。
- `synchronous=NORMAL`：在 WAL 下平衡可靠性和写入性能。
- `foreign_keys=ON`：每个连接统一启用外键约束。
- 业务继续保持短事务，LLM、TTS、ffmpeg 和 librosa 处理期间不持有数据库事务锁。

可继续使用 SQLite 的条件：

- 单台 FastAPI 实例。
- 同时在线约 10 人，持续写入频率低于每秒数次。
- 数据库文件和 `STORAGE_DIR` 位于同一台机器的本地 SSD，不放网络共享盘。
- 每日备份数据库和媒体生成目录。

出现以下任一信号时切换 Postgres：

- 需要两个及以上 FastAPI 实例。
- 持续出现写锁等待或 `database is locked`。
- 同时编辑人数稳定超过 10–20 人。
- 需要按用户、项目做复杂统计、检索或长事务。
- 需要托管平台的滚动升级、故障转移或跨机器任务执行。

## 3. Postgres 切换成本

领域模型基于 SQLModel，大部分 CRUD 可复用；主要迁移成本在数据库迁移体系和运维，而不是业务接口。

切换前必须完成：

1. 引入 Alembic，替换当前依赖 SQLite PRAGMA 和手写 `ALTER TABLE` 的迁移执行器。
2. 增加 PostgreSQL 驱动并验证 JSON、布尔值、索引和时间字段。
3. 编写 SQLite 到 Postgres 的一次性数据迁移和校验脚本。
4. 将测试矩阵扩展为 SQLite 快速回归和 Postgres 集成回归。
5. 配置连接池、备份恢复和生产密钥。

预估为独立的 2–4 个开发日，不应夹在当前自动粗剪迭代中实施。

## 4. 文件与对象存储

当前文件分为四类：

| 类型 | 当前位置 | 首发策略 |
|---|---|---|
| 原始视频/照片 | 用户配置的本地素材根目录 | 保持本地，不上传 |
| BGM 与 TTS 音频 | `STORAGE_DIR/audio`、`STORAGE_DIR/voiceover` | 本机持久化并备份 |
| 媒体预览缓存 | `STORAGE_DIR/media-preview-cache` | 可删除、可重建，不纳入长期备份 |
| 剪映草稿 | 用户本机剪映草稿目录 | 保持本地，由用户用剪映打开 |

现在不接对象存储，原因是剪映和 ffmpeg 最终仍需要本地文件路径。只把文件上传云端而没有桌面同步/下载缓存，会增加传输成本却不能解决剪映消费问题。

对象存储的接入触发条件：

- 服务部署到远程服务器，朋友需要跨设备访问同一项目。
- 需要云端保存 BGM、TTS、代理视频或成片预览。
- 单机磁盘无法满足容量、备份或可用性要求。

届时采用“对象存储保存源文件和代理文件 + 桌面端按需下载缓存 + 数据库只保存对象 key”的三层结构，不在数据库保存二进制内容。

## 5. 后台任务队列

当前 LLM 任务已持久化状态并由进程内线程执行，10 用户轻量使用下暂不引入 Redis/Celery。

保留当前方案的条件：

- 单进程部署。
- 同时执行的音频分析和 Vision 分析较少。
- 服务重启后允许任务标记为中断并由用户原参数重试。

引入外置队列的触发条件：

- 需要多实例部署或任务跨重启继续执行。
- 同时存在 3 个以上 ffmpeg/librosa/Vision 重任务并造成 CPU、内存拥塞。
- 需要任务优先级、定时任务、限流或独立 Worker 扩容。

推荐迁移顺序为 Postgres 任务状态表，再引入 Redis + RQ/Dramatiq；当前规模不需要 Celery 的完整复杂度。

## 6. 首发部署基线

- 一台 Windows 主机或个人工作站，建议 4 核 CPU、16 GB 内存、SSD。
- FastAPI 单进程运行；不要开启多个 Uvicorn worker 共同写同一个 SQLite 文件。
- Next.js 可与 FastAPI 同机运行，通过同源代理访问后端。
- `APP_SECRET_KEY` 使用稳定随机值；生产关闭 `LLM_OAUTH_MOCK`。
- 安装 ffmpeg，并验证 Edge TTS 所需网络连接。
- 每日备份 `video_sop.db`、`STORAGE_DIR/audio` 和 `STORAGE_DIR/voiceover`。
- 预览缓存不备份；剪映草稿和原始素材按用户现有本地备份方式管理。

## 7. 验收指标

- 10 个浏览器会话可并行读取项目，不出现数据库锁错误。
- 多个短事务写入发生冲突时可在 5 秒内等待恢复。
- LLM/TTS 失败不会丢失业务输入，可按原参数重试。
- 服务重启后运行中任务标记为中断，不误显示为仍在执行。
- 数据库与不可再生音频文件具备每日备份方案。
