# 二期执行清单

## 1. 文档目的

本文档用于把二期工作拆成可跟踪事项。优先级分为 `P0 / P1 / P2`，状态建议后续按“未开始 / 进行中 / 已完成”维护。

## 2. 当前状态概览

- 一期主流程：已可用
- 二期孤立能力：已起步
- 二期系统化整理：待继续推进

## 3. P0 必做项

### 3.1 文档与编码统一

- [x] 全部核心文档统一为 UTF-8
- [x] 避免 PowerShell / 编辑器编码不一致导致再次污染
- [x] 后续新增文档沿用当前编码规范

### 3.2 服务分层整理

- [x] 继续拆分 `repository` 中的生成型逻辑
- [x] 将节奏、主题、分镜、导出建议逐步下沉到独立 service
- [x] 形成清晰的“数据访问 / 规则生成 / LLM 调用”三层边界

### 3.3 数据结构与迁移规范

- [x] 明确数据库 schema 变更流程（`schemaversionentity` + 编号 migration）
- [ ] 每次新增字段都同步 migration、schema、api 文档
- [ ] 给音频分析、登录用户、LLM 配置相关数据结构补长期方案

### 3.4 基础稳定性

- [x] 继续排查工作流页面之间的可点击状态联动问题
- [x] 统一成功提示、加载提示、错误提示口径
- [ ] 关键操作补最小回归检查

## 4. P1 优先功能项

### 4.1 音频上传与节拍识别

- [x] 扩展音频格式支持，不只限于当前基础格式
- [x] 提高 beat point 识别质量（librosa 主识别 + 能量算法兜底）
- [x] 明确“规则生成”和“音频分析生成”的来源标记（含 rule_fallback）
- [x] 让节奏说明更好解释识别结果（BPM、时长、节拍点数量）
- [x] 识别失败时自动回退规则生成
- [x] 支持移除已绑定音频 / 重新上传时清理旧文件

### 4.2 LLM 主题建议

- [ ] 强化主题候选的可解释性
- [ ] 增加候选间差异度控制
- [ ] 明确失败兜底和超时兜底策略

### 4.3 LLM 分镜建议

- [ ] 在现有节奏、素材、主题基础上输出更可信的分镜建议
- [ ] 后续支持“大模型建议顺序”作为二层排序来源之一
- [ ] 强化素材绑定和地点连续性校验

### 4.4 LLM 导出建议

- [ ] 让标题、标签、文案建议更贴近平台风格
- [ ] 预留后续多平台差异化策略
- [ ] 将最终导出结果更完整写回时间线脚本

### 4.5 用户与鉴权

- [x] 将用户信息从前端代码迁移到后台数据库
- [x] 改为后端校验登录
- [ ] 保留新增用户入口的数据结构
- [ ] UI 暂时仍可只开放导演账号

### 4.6 LLM Provider 与鉴权标准化

- [ ] 建立 Provider Registry
- [ ] 建立统一 Auth Service
- [x] 支持 API Key 配置
- [ ] 预留 OAuth / Device Code 能力
- [ ] 不支持通过抓取网页登录 Cookie 作为正式方案

## 5. P2 增强项

### 5.1 更强业务校验

- [ ] 检查分镜总时长与目标时长偏差
- [ ] 检查未绑定素材的镜头
- [ ] 检查跨地点跳切风险
- [ ] 检查导出文案与主题一致性

### 5.2 导出能力增强

- [ ] 扩展导出内容结构
- [ ] 增加更多下载格式
- [ ] 优化预览阅读体验

### 5.3 交互体验优化

- [ ] 持续统一列表页与编辑页交互模式
- [ ] 优化复杂选择器的搜索、分组、键盘操作
- [ ] 保持所有时间类输入控件体验一致

### 5.4 AI 深化能力

- [ ] 引入更强的大模型分镜组织能力
- [ ] 研究更合理的镜头复用策略，但不一定进入标准主流程
- [ ] 评估后续自动粗剪的结构准备

## 6. 推荐推进顺序

1. 先完成 P0
2. 再做音频节拍识别和 LLM 核心能力增强
3. 接着完成用户与鉴权后端化
4. 最后补齐 P2 的增强项
